JavaScriptdagi xatolar turlari

1. Syntax Error (Sintaksis xatosi)

Kod JavaScriptning sintaksis qoidalariga mos kelmasa yuzaga keladi.

let x = ; // Xato: qiymat ko'rsatilmagan console.log('Salom); // Xato: yopiluvchi qo'shtirnoq yetishmayapti

2. Reference Error (Manba xatosi)

Belgilangan o'zgaruvchi yoki funksiyaga murojaat qilishda ular aniqlanmagan bo'lsa yuzaga keladi.

console.log(myVar); // Xato: myVar aniqlanmagan

3. Type Error (Tur xatosi)

Ma'lumot turlari mos kelmagan operatsiya bajarilganda yuzaga keladi.

let x = null; x.toString(); // Xato: null qiymatida metodni chaqirib bo'lmaydi

4. Range Error (Oraliq xatosi)

Qabul qilinadigan oraliqdan tashqaridagi qiymat ishlatilganda yuzaga keladi.

let arr = new Array(-1); // Xato: massivning o'lchami manfiy bo'lishi mumkin emas

5. Eval Error (Eval xatosi)

eval funksiyasidan noto'g'ri foydalanilganda yuzaga keladi. Zamonaviy versiyalarda kam uchraydi.

eval("alert('Salom')"); // Ishlaydi, lekin noto'g'ri qo'llanilganda xatolarga olib kelishi mumkin

6. URI Error (URI xatosi)

encodeURI yoki decodeURIComponent funksiyalariga noto'g'ri ma'lumot uzatilganda yuzaga keladi.

decodeURIComponent('%'); // Xato: noto'g'ri URI qismi

7. Aggregate Error (Yig'ma xato)

Bir nechta xatolarni guruhlash uchun ishlatiladi, masalan, Promise.all holatida.

Promise.all([ Promise.reject('Xato 1'), Promise.reject('Xato 2') ]).catch(err => console.log(err)); // AggregateError: bir nechta xatolarni o'z ichiga oladi

Xatolarni boshqarish

JavaScriptda xatolarni boshqarish uchun try...catch konstruktsiyasidan foydalaniladi.

try { let result = x + 10; // ReferenceError } catch (error) { console.log('Xato:', error.message); }
